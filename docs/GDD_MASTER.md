# 1. Видение проекта

## 1.1. Высокоуровневая идея
«Дао Бессмертного» — это вертикальная онлайн-игра по мотивам аниме/сянься-эстетики, где игрок проходит путь от начинающего культиватора до бессмертного, развивая персонажа, техники, духовное оружие, питомцев, орден, домен, социальные связи и место в мировом рейтинге.

Игра должна сочетать:
- эмоциональную fantasy power progression;
- удобный мобильный one-hand интерфейс;
- постоянный онлайн-прогресс;
- сильную мета-игру между боями;
- социальные системы: ордена, чаты, совместные активности, PvP и межсерверные события;
- idle/offline прогресс без перегруза ручным гриндом.

## 1.2. Фантазия игрока
Игрок должен чувствовать:
- что он действительно восходит по ступеням Дао;
- что каждый день становится сильнее;
- что его персонаж визуально и системно меняется;
- что мир вокруг живой: события, письма, миссии, награды, боссы;
- что интерфейс богато оформлен, но остаётся читаемым и быстрым.

## 1.3. Основные pillars проекта
1. **Путь культивации**  
   Центральная фантазия — прорывы по стадиям, рост духовной силы, раскрытие новых механик.
2. **Вертикальный мобильный comfort UX**  
   Все ключевые действия доступны в 1–3 тапа, интерфейс читается на маленьком экране.
3. **Онлайн-жизнь мира**  
   Чаты, ордена, асинхронное и синхронное PvP, рейтинги, мировые события.
4. **Idle + active hybrid**  
   Игрок получает прогресс оффлайн, но в онлайне есть важные активности, влияющие на рост.
5. **Эстетика xianxia премиум-класса**  
   Нефрит, золото, туман, горы, пагоды, каллиграфия, духовные артефакты, мягкие свечения.

# 2. Анализ визуального референса

## 2.1. Что видно в референсе
Референс демонстрирует:
- центрального героя в полный рост;
- большую «геройную» домашнюю сцену вместо плоского меню;
- насыщенный верхний статус-бар;
- боковые вертикальные кнопки функций;
- нижнюю панель основных разделов;
- крупную центральную CTA-кнопку культивации;
- живой фон с небесными горами и световыми потоками;
- нефритово-золотую эстетическую доминанту;
- плотный, но иерархичный интерфейс.

## 2.2. Что из референса берём как основу
Берём:
- герой по центру как эмоциональный anchor;
- лобби как «живую сцену», а не список кнопок;
- декоративные рамки и восточные узоры;
- зелёно-золотой акцент;
- боковые функциональные иконки;
- верхнюю ресурсную строку;
- нижнюю таб-навигацию;
- культивацию как главную кнопку-смысл.

## 2.3. Что улучшаем по UX
Чтобы не перегрузить пользователя:
- убираем дублирующиеся блоки иконок;
- делаем строгую иерархию приоритетов;
- часть функций переносим в раскрывающиеся панели;
- сохраняем визуальную роскошь, но уменьшаем шум;
- усиливаем читаемость типографики;
- используем понятные подписи и статусные индикаторы.

# 4. Высокоуровневый gameplay loop

## 4.1. Macro loop
1. Игрок входит в игру.
2. Собирает оффлайн-награды и ресурсы культивации.
3. Усиливает персонажа: культивация, навыки, экипировка, артефакты, питомцы.
4. Проходит контент: сюжет, башня, карта, PvP, орден.
5. Получает ресурсы.
6. Совершает прорыв / апгрейды.
7. Возвращается позже за новым циклом прогресса.

## 4.2. Session loop
1. Login / inbox / rewards  
2. Collect idle gain  
3. Daily missions  
4. Main progression activity  
5. Upgrade pass  
6. Social / guild / events  
7. Exit with offline accumulation enabled

## 4.3. Emotional loop
- ожидание награды;
- усиление;
- визуальный рост;
- прорыв;
- социальное признание;
- редкий дроп;
- закрепление статуса.

# 5. Основные игровые системы

## 5.1. Система культивации
Центральная система прогрессии персонажа.

### 5.1.1. Иерархия стадий
- Смертный
- Очищение тела
- Сгущение Ци
- Заложение Основания
- Золотое Ядро
- Зарождающаяся Душа
- Формирование Духа
- Пустота
- Полубессмертный
- Бессмертный
- Небесный Бессмертный
- Дао-Владыка

Каждая стадия содержит подранги:
- Ранний этап
- Средний этап
- Поздний этап
- Пик

### 5.1.2. Ресурсы культивации
- Духовные камни
- Эссенция Ци
- Камни прорыва
- Свитки прозрения
- Сердца демонов / материалы босса
- Небесная удача / premium-ресурс для безопасного прорыва

### 5.1.3. Механика прогресса
- игрок получает культивационный опыт;
- при заполнении шкалы открывается «Прорыв»;
- прорыв может требовать условия:
  - минимальная сила;
  - материалы;
  - пройденный сюжет;
  - победа над испытанием;
  - определённый уровень техник.

### 5.1.4. Типы прорыва
- Гарантированный
- Рискованный
- Усиленный (с бустерами)
- Авто-прорыв

### 5.1.5. Последствия прорыва
- рост базовых статов;
- открытие новых навыков;
- открытие слотов артефактов;
- доступ к новым режимам;
- новая визуальная аура;
- новые титулы / рамки / VFX.

## 5.2. Боевая система
Рекомендуемый формат: **асинхронный полуавтоматический бой с возможностью ручного прожима навыков**.

### 5.2.1. Почему такой формат
- подходит вертикальному мобильному UX;
- хорошо сочетается с idle-прогрессом;
- не требует сложного real-time управления;
- легко масштабируется под PvE, PvP, башни, боссы;
- даёт место для стратегии через состав билдов.

### 5.2.2. Базовая модель боя
Отряд игрока:
- главный персонаж;
- дух-компаньон / питомец;
- призывы / духовные мечи / спутники.

### 5.2.3. Базовые боевые параметры
- HP
- ATK
- DEF
- Speed
- Crit Rate
- Crit Damage
- Hit
- Dodge
- Break Power
- Tenacity
- Qi Recovery
- Skill Power
- Element Bonus
- Element Resist

### 5.2.4. Стихии
- Дерево
- Огонь
- Земля
- Металл
- Вода
- Молния
- Тьма
- Свет

### 5.2.5. Навыки
Типы:
- обычная атака;
- активный навык;
- ультимейт;
- пассивка;
- аура;
- резонанс с оружием;
- пробуждённый навык стадии.

### 5.2.6. Боевой темп
- бой mostly auto;
- игрок может:
  - включить авто;
  - назначить приоритет навыков;
  - в manual-режиме нажимать ультимейты;
  - ускорять бой x1/x2/x3;
  - пропускать анимации в фарм-контенте.
